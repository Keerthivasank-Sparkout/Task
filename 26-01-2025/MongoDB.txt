1.What is Database?
    >A database is an organized collection of structured information or data, typically stored electronically in a computer system and controlled by a Database Management System (DBMS). 
    It enables efficient data storage, retrieval, modification, and management, often using SQL for structured data or other formats for NoSQL types. 

2.what is MongoDB?
    >MongoDB is a popular, open-source NoSQL document database designed for high performance, flexibility, and scalability. 
    >Unlike traditional relational databases that use tables and rows, it stores data in JSON-like documents (BSON) with dynamic schemas. 
    >This allows developers to store various data structures, making it ideal for modern, rapidly evolving applications.

>Logical Operators
    .$and
    .$or
    .$nor

            db.<collectionName>.find({$and[{name:"keerthi"},{age:24}]}) - only two condition matched data only return.

            db.<collectionName>.find({$nor[{name:"keerthi"},{age:24}]}) - it skip the give data in query then return the all data in entire collection.

>Validation
    >Schema validation rules can be created in order to ensure that all documents in a collection share a similar structure.
     Ex:
        db.createCollection("<collectionName>",{
            validator:{
                $jsonSchema:{
                    bsonType:"object",
                    required:["name","age"],
                    properties:{
                        name:{
                            bsonType:"string",
                            description:"name is required"
                        },
                        age:{
                            bsonType:"number",
                            description:"age is number - required"
                        }
                    }
                }
            }
        })
>lookup
    This aggregation stage performs a left outer join to a collection in the same database.
        There are four required fields:
            .from: The collection to use for lookup in the same database
            .localField: The field in the primary collection that can be used as a unique identifier in the from collection.
            .foreignField: The field in the from collection that can be used as a unique identifier in the primary colle
            .as: The name of the new field that will contain the matching documents from the from collection.
